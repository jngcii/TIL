# Intro

###### 2020.06.25

## I. 스프링 프레임워크
- Spring framework는 주요 기능으로서 DI, AOP, MVC, JDBC 등을 제공한다.

## II. 스프링에서 제공하는 모듈
  | 모듈 | 기능 |
  | --- | --- |
  |spring-core | 스프링의 핵심인 DI를 제공|
  |spring-aop | AOP 구현 기능 제공|
  |spring-jdbc | 데이터베이스를 쉽게(적은 양의 코드로) 다룰 수 있는 기능 제공|
  |spring-tx | 스프링에서 제공하는 트랜잭션 관련 기능 제공 |
  |spring-webmvc | 스프링에서 제공하는 컨트롤러(Controller)와 뷰(View)를 이용한 스프링MVC 구현 기능 제공 |
  - 스프링 프레임워크에서 제공하고 있는 모듈을 사용하려면, 모듈에 대한 의존 설정을 개발 프로젝트에 XML 파일 등을 이용해서 개발자가 직접 하면 된다.

## III. 스프링 컨테이너
- 스프링에서 객체를 생성하고 조립하는 컨테이너로, 컨테이너를 통해 생성된 객체를 빈(Bean)이라고 부른다.
- XML문서 등을 통해 객체 생성 및 속성 데이터 작성을 하고, 그것을 스프링 컨테이너에서 만들어진 객체(Bean)을 조립해서 애플리케이션 구현
- 스프링 컨테이너에 모든 객체가 있고 그 객체 안에는 내가 필요한 모든 기능이 들어있다. 그 기능을 필요할때마다 꺼내서 내가 원하는 기능을 구현하면 된다.
